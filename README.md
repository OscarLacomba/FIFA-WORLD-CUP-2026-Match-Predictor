## Features

- **Match Prediction** — predicts Home Win / Draw / Away Win with confidence scores
- **27 engineered features**:
  - Elo ratings — difference and ratio (pre-tournament snapshot for all 48 qualified teams)
  - Recent form — points and goal-difference averages over last 5 and 10 matches
  - Win streaks
  - Days since last match (fatigue/inactivity)
  - Head-to-head historical record between each pair of teams
  - Elo × form interaction
  - World Cup flag (competitive vs friendly matches)
  - Home/away differential features
- **Competitive-matches training** — model trained only on competitive fixtures (qualifiers, World Cups, continental cups), excluding friendlies which distort home-advantage signal
- **Walk-forward validated** — accuracy stable (0.579-0.593) across 7 different temporal train/test splits
- **Monte Carlo Simulation** — simulates the full tournament 1,000 times with precomputed pairwise probabilities
- **Web App** — 4-tab Gradio interface: Predictor, Matches, My Prediction, Leaderboard
- **SQLite Database** — stores users, matches and predictions persistently
- **AI Analysis** — Claude API generates natural language match explanations (with statistical fallback)
- **Leaderboard** — real-time ranking of users vs AI model

## Usage

### 1. Train the model

Run the feature-engineering and training cells in the notebook. This will:
- Download the datasets via `kagglehub`
- Engineer 27 features (Elo diff/ratio, recent form, streaks, head-to-head, rest days, Elo×form, World Cup flag)
- Filter out friendly matches from the training set
- Train XGBoost on competitive matches only
- Validate with walk-forward across 7 temporal splits
- Save the model to `wc2026_model.pkl`

### 2. Build the database

Run the SQLite cell to create `wc2026.db` with the World Cup 2026 group-stage fixtures.

### 3. Launch the web app

Run the Gradio cell. It will output a public URL:
```
Running on public URL: https://xxxxxx.gradio.live
```

### 4. Scoring system

| Event | Points |
|-------|:------:|
| Correct result (Win/Draw/Loss) | +3 |
| Exact score bonus | +2 |
| Wrong prediction | 0 |

## Model Performance

| Model | Accuracy | F1-macro |
|-------|:--------:|:--------:|
| Logistic Regression (all matches) | ~0.527-0.577 | — |
| Random Forest (all matches) | ~0.539-0.585 | — |
| Voting / Stacking ensembles | ~0.587 | — |
| Two-stage (draw classifier + home/away) | ~0.589-0.594 | — |
| **XGBoost — competitive matches only (final model)** | **0.5933** | **0.4439** |

### Why 0.59?

Predicting football results has a well-documented accuracy ceiling of roughly 50-62% with tabular statistical features — our final model sits at the upper end of that range. This was confirmed by:

- **Walk-forward validation** across 7 different train/test cutoff years (2015-2021): accuracy ranged 0.579-0.593, confirming stability
- **Multiple ensemble techniques** (Voting, Stacking, two-stage draw/win classifiers) all converged to ~0.587-0.594, indicating the limit is informational, not algorithmic
- **Filtering friendlies out of training** was the single most effective improvement (+0.5-1.0 pp), since competitive matches have a more consistent home-advantage signal

**Context for the result:**
- vs. random baseline (33.3%): **+78% relative**
- vs. "always predict Home Win" (46.9-50.4% depending on subset): **+18-26% relative**
- Consistent with published academic benchmarks for football outcome prediction (50-62%)

### Top features by importance

1. `elo_diff` / `elo_ratio` — Elo rating difference and ratio between teams
2. `diff_pts_5` / `diff_pts_10` — recent form differential (points per game)
3. `diff_gd_5` / `diff_gd_10` — recent goal-difference differential
4. `home_elo` / `away_elo` — absolute team strength
5. `elo_x_form` — interaction between Elo gap and recent form
6. `h2h_home_avg_pts` — historical head-to-head record
7. `home_win_streak` / `away_win_streak` — momentum

## Deploy to Hugging Face

1. Create a new Space at [huggingface.co/new-space](https://huggingface.co/new-space) — SDK: **Gradio**
2. Make sure your Space `README.md` header looks like:
   ```yaml
   ---
   title: WC2026 Match Predictor
   emoji: ⚽
   colorFrom: blue
   colorTo: green
   sdk: gradio
   app_file: app.py
   pinned: true
   ---
   ```
   (omit `sdk_version` to avoid version-mismatch errors)
3. Upload these files:
   - `app.py`
   - `requirements.txt`
   - `wc2026_model.pkl`
   - `wc2026_features.pkl`
   - `wc2026.db` (optional — created automatically on first run)
4. Your app will be live at `https://huggingface.co/spaces/YOUR_USER/wc2026-match-predictor`
